How it works

1
Add Sources Point the tool at RSS feeds or web pages you want to monitor. Sources are fetched every time you run the pipeline.
2
Set Keywords (optional) Only items whose title or summary contains at least one keyword will be saved. Leave blank to capture everything.
3
Run the Pipeline Fetches all sources, applies your keyword filter, deduplicates against previous runs, and saves new items.
4
Export Results Download a CSV of all saved items at any time. The pipeline also auto-generates an XLSX report and an HTML site.
Keywords Filters what gets saved during a pipeline run
How keywords work: When you run the pipeline, only items whose title or summary contains at least one of these keywords are saved to the data file. Items that don't match are discarded.
  • Leave blank to save all items from every source.
  • Keywords are case-insensitive — "Bitcoin" matches "bitcoin".
  • Changing keywords only affects the next run — previously saved items are not removed.

Sources {{ sources | length }} configured
Sources are the feeds and pages the pipeline fetches from. Two types are supported:
  • RSS — provide the feed URL; items are parsed automatically.
  • Scrape — provide a web page URL plus CSS selectors that tell the tool where to find item titles, links, dates, and summaries on the page.
Items are deduplicated by URL — re-running the pipeline won't create duplicates.

Export Downloads from the current saved data
The CSV contains all items saved so far — everything that has passed the keyword filter across all pipeline runs. Columns: source, title, url, published, summary, created_at.

To reset and start fresh: delete items.json from the project folder, then re-run the pipeline. The next run will fetch everything as if it were the first time.
The pipeline also auto-generates report.xlsx and an HTML site in the site/ folder every time it runs.
