How it works
1

Add Sources

Point the tool at RSS feeds or web pages you want to monitor. Sources are fetched every pipeline run.

2

Set Keywords

Only items whose title or summary contains a keyword are saved. Leave blank to capture everything.

3

Run the Pipeline

Fetches all sources, applies your filter, deduplicates, and saves new items. Runs automatically every 4 hours.

4

Export Results

Download a CSV of all saved items — all sources together, or one source at a time — at any time.

Keywords
Filter what gets saved during each pipeline run
How keywords work: When the pipeline runs, only items whose title or summary contains at least one keyword are saved. Items that don't match are discarded.
  • Leave blank to save all items from every source.
  • Keywords are case-insensitive — "Bitcoin" will match "bitcoin".
  • Changing keywords only affects the next run — previously saved items are not removed.

  • RSS — paste the feed URL; items are parsed automatically.
  • Scrape — paste a web page URL, then provide CSS selectors to tell the tool where titles, links, and dates are on the page.
Items are deduplicated by URL — running the pipeline multiple times won't create duplicates.

Export
Download your collected data at any time
The CSV contains all items saved so far — everything that has passed the keyword filter across all pipeline runs. Columns: source, title, url, published, summary, created_at.

To export only one source, use the CSV button next to that source in the table above.

To start fresh: delete items.json from the project folder and re-run the pipeline.
The pipeline also auto-generates report.xlsx and an HTML site in the site/ folder on every run.
